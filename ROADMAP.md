# CassieProject Roadmap

## 当前阶段

本地优先基础、“预算—目标—复盘”决策闭环和正式专项已经建立；逐笔收入已改为月度可支配金额，当前进入真实使用观察期，先验证支出记账、预算和月结是否足够顺手，再继续增加专项经验复用功能。

- 建立项目规范和路线图。
- 将金额改为整数分。
- 使用 UUID 标识账目。
- 增加数据结构版本、校验和旧数据迁移。
- 修复专项详情统计范围和日历年／月状态混用。
- 增加账目编辑、删除撤销和安全导入。
- 移除动态内联事件处理器，完善存储错误反馈。
- 将项目规范统一为 `AGENTS.md`，主账本页面统一为静态站点入口 `docs/index.html`。
- 增加 schema v2 演示账本 `test-data/account-book-demo-v2.json`，用于导入和界面效果测试。
- 增加 GitHub Pages PWA 资源：manifest、Service Worker 和本地应用图标。
- 移除 Google Fonts，`docs/` 运行时不再依赖第三方网络资源。
- 增加备份时间记录、未备份／过期提醒和导入数据摘要。
- 增加异常数据写入锁定、原始内容下载和有效记录重建流程。
- 增加最近使用和本地高频分类；旧版收藏数据继续兼容读取和备份，但不再要求用户手动维护收藏。
- 保留旧版快速模板数据的读取、备份和导入兼容，但移除低频模板入口，首页只保留“复制最近一笔”。
- 增加复制最近一笔、复制明细账目和“保存并再记”。
- 将明细筛选收敛为关键词、收支类型和日期范围，并保留结果汇总。
- 增加返回本月快捷入口，筛选结果自动展开到明细。
- 增加完整备份 v1，统一导出账目、最近分类、收藏分类和快速模板。
- 增加“合并账目”“覆盖账目”和“完整恢复账目与设置”，继续兼容 schema v2 和旧数组备份。
- 为快捷设置增加独立版本和更新时间，设置变化后会触发重新备份提醒。
- 增加 decisions v1 独立规划数据；当前界面按月份只维护一个日常总预算，旧分类预算继续兼容保存和导入。
- 增加“规划”页面、概览预算执行卡片、80％预警、100％超支和复制上月预算。
- 将完整备份升级到 v2，纳入规划数据并继续兼容完整备份 v1。
- 增加财务目标，支持应急储备、旅行、教育、偿债和自定义类型。
- 增加独立的目标投入记录、累计进度、剩余金额和按目标日期计算的建议月投入。
- 增加目标编辑、暂停、恢复、完成和重新启用，目标及投入随规划数据进入完整备份。
- 增加月度复盘，汇总可支配金额、支出、剩余、预算执行和目标投入，并生成基于本地数据的观察。
- 将月度复盘收敛为“一个结论＋下个月一件事”；旧版额外行动继续兼容保留，上月行动会在次月自动出现并可标记完成。
- 禁止提前填写未来月份复盘，复盘和行动状态随规划数据进入完整备份。
- 增加目标投入明细，支持查看每笔投入的日期、金额和备注。
- 增加目标投入编辑和二次确认撤销，修改后同步更新目标进度与对应月份复盘，并在保存失败时回滚。
- 增加最近三个完整月份的现金流能力分析，排除尚未结束的本月数据。
- 将房租、水电燃气识别为固定承诺，保留最近三个完整月份的现金流基础计算，但移除缺乏实际账户余额支撑的 50％／70％／90％目标投入情景。
- 联合本月可支配金额、日常预算、当月开始的专项预算和进行中目标的建议月投入，在预算页只提示一个计划后结余或缺口，并标明估算边界。
- 增加具有稳定 ID 的正式专项，支持旅行、装修、节日、医疗、教育、搬家和其他类型，并记录预算、日期、参与人数和进行中／完成状态。
- 增加当前专项，新建首个专项后自动设为当前；只有记账日期位于当前专项日期范围内时才自动带入，专项完成后自动取消当前状态。
- 保留住宿、交通、餐饮等支出分类，同时按专项汇总总支出、预算执行、记录数、人均和人均每天费用。
- 将正式专项支出从日常总预算和分类预算中排除，避免生活事件挤占日常消费额度；总支出统计仍包含全部支出。
- 将开始日期位于本月的专项预算纳入现金流计划，并在完整备份 v2 中保存专项定义、当前专项和账目关联。
- 继续兼容旧版专项标签和不含专项字段的旧备份；旧记录不自动改写，新记录使用 `projectId` 关联正式专项。
- 修正当月和当年同比口径：当前周期只与上月／上年同日之前的数据比较，历史周期继续比较完整周期。
- 将一级导航收敛为“首页、明细、计划”，日历合并进明细，月度复盘与现金流判断合并进“月结”。
- 简化记账表单：优先显示最近／高频分类，日期、备注和专项按需展开；保存前明确显示分类和专项归属，并直接提供“保存并再记”。
- 简化首页和数据管理：首页只保留复制最近一笔、日常预算、当前专项和一个观察，趋势与排行默认折叠；完整备份与导入统一放入带文字标识的数据管理弹窗。
- 简化预算和专项指标：预算只维护日常总额；专项预算改为选填，只有旅行专项显示参与人数、人均和人均每天，已完成专项默认折叠。
- 明确财务目标用于留出资金、正式专项用于归集实际花费；目标编辑和状态操作集中到管理菜单，已暂停和已完成目标默认折叠。
- 日历和专项详情中的账目支持直接打开复制、编辑和删除菜单，减少返回列表的绕行。
- 弹窗增加背景锁定、焦点循环和关闭后焦点恢复，手机端保持 390px 视口无横向溢出。
- 允许页面缩放，并将主要导航、分类选择、弹窗操作、记录菜单和删除撤销等高频触控目标调整到至少 44px。
- 修复手机端打开记账弹窗时金额输入框自动聚焦导致的视口放大：初始焦点改到关闭按钮，并确保手机端表单控件字号不低于 16px。
- 修复日历每日金额直接把整数分当成元展示的问题；每日支出继续按整数分聚合，格子统一转换为元后再紧凑显示。
- 取消逐笔收入的新增、复制、编辑和删除入口；新账目只允许记录支出，旧收入在明细和导入导出中只读兼容。
- 为月度计划增加选填的可支配金额，旧预算在没有该字段时继续兼容读取。
- 将首页、月结和现金流判断改为使用可支配金额；现金流不再依赖历史月均收入，缺少必要计划数据时不输出误导性的结余判断。
- 将日历收敛为每日支出视图；旧收入仍可通过明细的“历史收入”筛选查看。
- 建立 `README.md` 当前使用说明，覆盖本地存储边界、记账、明细、预算、专项、目标、月结、备份恢复和数据救援，并在项目规范中要求用户流程变化时同步维护。
