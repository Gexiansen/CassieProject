# CassieProject Roadmap

## 当前阶段

本地优先基础阶段已完成：单人手机端账本具备版本化数据、JSON 迁移、PWA 离线壳和数据救援能力。

## 已完成

- 完成 `docs/index.html` 的静态代码审查。
- 明确家庭共享需要独立身份、权限、同步和冲突处理，不能直接以 `localStorage` 作为共享数据源。
- 建立项目规范和路线图。
- 将金额改为整数分。
- 使用 UUID 标识账目。
- 增加数据结构版本、校验和旧数据迁移。
- 修复专项详情统计范围和日历年／月状态混用。
- 增加账目编辑、删除撤销和安全导入。
- 移除动态内联事件处理器，完善存储错误反馈。
- 将项目规范统一为 `AGENTS.md`，主账本页面统一为静态站点入口 `docs/index.html`。
- 增加 schema v2 演示账本 `test-data/account-book-demo-v2.json`，用于导入和界面效果测试。
- 增加 GitHub Pages PWA 资源：manifest、Service Worker 和本地应用图标。
- 移除 Google Fonts，`docs/` 运行时不再依赖第三方网络资源。
- 增加备份时间记录、未备份／过期提醒和导入数据摘要。
- 增加异常数据写入锁定、原始内容下载和有效记录重建流程。

## 最近验证

- `docs/index.html` JavaScript 静态语法检查通过，动态内联 `onclick` 数量为 0。
- 在 390×844 手机视口完成新增支出、含单引号专项名称、专项详情、编辑、删除撤销、年度切入日历和刷新持久化验证，浏览器控制台无报错。
- 旧版数据迁移验证通过：12.34 元迁移为 1234 分，旧 ID 转为稳定的 `legacy-*`，并写入 schema v2。
- 导入记录的日期、分类、金额和 ID 校验已通过独立函数验证；系统文件选择器未纳入自动化测试。
- 演示账本通过应用自身校验：45 笔记录全部有效，2026 年 7 月包含 18 笔收支和 2 个专项标签。
- PWA 资源语法和引用检查通过；关闭本地 HTTP 服务后，页面及本地账本仍可离线重新加载。
- 备份状态的空账本、从未备份、刚备份和超过 14 天四种分支验证通过。
- 正常、部分异常和无法解析三种本地数据验证通过；异常数据会保留原始内容并锁定写入。
- 当前目录已初始化 Git 仓库，阶段一代码已纳入 `main` 分支版本管理。
