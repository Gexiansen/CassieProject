# 我的小账本

一个通过 GitHub Pages 使用的手机端本地支出记账工具。它通过“支出—预算—目标—月结”帮助个人或家庭成员在各自设备上做财务决策。

主账本入口是 `docs/index.html`。`allen_account.html` 是独立的儿童玩具钱包，不与主账本共享数据，本说明不涵盖该页面。

## 使用前必须知道

- 软件没有账号、服务器和网络同步，账目、设置、预算、专项、目标及月结都保存在当前浏览器中。
- 同一部手机上的不同浏览器、普通模式与无痕模式拥有不同的数据空间。
- 清除网站数据、卸载浏览器或更换设备，可能导致本地数据丢失。
- 请定期通过右上角“数据管理”导出完整 JSON 备份，重要操作前也建议先备份。
- 后续即使家庭成员多人使用，也优先由每个人在自己的设备上独立记账，再通过备份文件迁移；当前没有自动合并家庭账本的功能。

## 首次使用

1. 用手机浏览器打开部署在 GitHub Pages 上的页面。
2. 点击右下角“＋ 记一笔”，记录第一笔支出。
3. 进入“计划—预算”，设置本月可支配金额和日常预算。
4. 使用一段时间后，在“计划—月结”查看本月结果并写下一个结论和下个月的一件事。
5. 点击右上角“数据”，导出第一份完整备份。

首次成功打开后，页面支持离线使用。为了更接近原生应用体验，可以在手机浏览器菜单中选择“添加到主屏幕”或“安装应用”。

## 页面结构

软件只有三个一级入口：

- “首页”：查看本期支出、可支配金额、预算进度、当前专项、消费观察、支出趋势和排行。
- “明细”：用列表或日历查看账目，并进行搜索、筛选、编辑、复制和删除。
- “计划”：管理预算、正式专项、财务目标和月结。

首页右上角的“月／年”用于切换统计周期，左右箭头用于切换月份或年份。当前月份和年份只与上月同期或上年同期比较，避免拿不完整周期与完整周期直接比较。

## 记录一笔支出

点击任意页面右下角“＋ 记一笔”：

1. 输入金额。
2. 从最近使用或本地高频分类中快速选择；需要其他分类时点击“更换”。
3. 如需修改日期、填写备注或关联专项，展开“日期、备注与专项”。
4. 保存前确认表单底部的分类和专项归属，再点击“保存”。

软件不再逐笔记录收入。金额在内部按整数分保存，界面统一按元显示。

### 快速记录

- 首页会提供“复制最近一笔”，适合重复发生的消费。
- 连续补记多笔账目时，可以直接点击“保存并再记”。

### 当前专项

存在“当前专项”且记账日期位于专项开始和结束日期之间时，新支出才会默认关联该专项。表单中会显示专项提示；点击提示里的“取消”，即可把本次支出改回普通日常支出。专项日期之外不会自动关联，但仍可手动选择。

导入或升级前已有的收入记录会继续保留在历史明细和备份中，但只能查看，不能复制、编辑或删除。

## 查看和管理明细

进入“明细”后，可以在“列表”和“日历”之间切换。

### 列表

- 账目默认按年份和月份折叠汇总。
- 点击年份或月份可以展开和收起。
- 每笔支出右侧的“⋯”提供复制、编辑和删除操作；历史收入只能打开查看。
- 删除后有 12 秒撤销时间。

### 搜索和筛选

搜索框可以查找备注、专项名称或分类。展开“更多筛选条件”后，还可以组合筛选：

- 支出或历史收入；
- 全部、本月、上月、今年或自定义日期。

筛选结果会显示记录数、历史收入和支出汇总。

### 日历

- 日历只显示每日支出。
- 格子中的金额是当天支出的合计，按元显示；较大金额会使用 `k` 或“万”紧凑展示。
- 点击日期可以查看当天的每笔支出和精确金额；再点击某笔支出，可以直接复制、编辑或删除。

## 设置日常预算

进入“计划—预算”：

1. 可选填写“本月可支配金额”，表示本月可以安排给支出和目标的钱。
2. 设置“本月日常预算”。
3. 点击“保存月度计划”。

如果上个月已经设置月度计划，可以复制上月的可支配金额和日常预算作为起点。

正式专项支出不会占用日常总预算。例如旅行期间的住宿、交通和餐饮仍保留原分类，但由旅行专项单独归集。

预算页会用本月可支配金额减去日常预算、本月开始的专项预算和进行中目标的建议投入，给出一个“计划后结余”或缺口。没有设置日常预算时，才会使用最近三个完整月份的历史平均支出作为参考；缺少可支配金额时不会给出结余判断。

## 管理正式专项

正式专项用于记录一次完整的生活事件，例如旅行、装修、节日、医疗、教育或搬家。

进入“计划—专项”，点击“＋ 新建专项”，填写：

- 专项名称和类型；
- 专项预算，选填；
- 开始和结束日期；
- 旅行参与人数，选填，仅旅行专项显示。

首个进行中的专项会自动成为当前专项。也可以在专项卡片中手动“设为当前”或“取消当前”。完成专项后，它不会再自动关联新支出。

专项页面会展示实际支出和记录数；设置预算后还会展示预算执行和剩余金额。旅行专项会计算人均和人均每天费用，便于以后规划类似旅行。支出明细可以直接点击管理，已完成专项默认折叠显示。

## 管理财务目标

进入“计划—目标”，可以建立应急储备、旅行、教育、偿债或自定义目标。

每个目标包含目标金额和目标日期，系统会根据剩余金额与剩余月份计算建议月投入。

目标只表示为未来留出资金；旅行、装修等实际花费应使用专项。当你实际留出一笔钱后，点击目标中的“记录投入”进行确认。目标投入：

- 不会自动生成支出记录；
- 可以查看、编辑和撤销每笔投入；
- 会同步影响目标进度和对应月份的月结汇总。

目标支持暂停、恢复、完成和重新启用。编辑、状态切换和投入记录纠正集中在“管理”中，已暂停和已完成目标默认折叠显示。

## 完成月结

进入“计划—月结”，可以查看：

- 本月可支配金额、支出和剩余金额；
- 日常预算执行情况；
- 目标投入汇总；
- 基于当前本地数据生成的本月观察。

月结只要求填写两项：

1. 本月最值得记住的一个发现；
2. 下个月只做的一件事。

上个月保存的行动会在下个月出现，并可以标记完成。未来月份不能提前填写月结。

## 备份和恢复数据

点击页面右上角“数据”进入“数据管理”。

### 导出完整备份

“导出完整备份”会生成 JSON 文件，包含：

- 全部支出记录及兼容保留的历史收入；
- 最近分类等快捷设置；
- 月度预算；
- 正式专项及当前专项；
- 财务目标和投入记录；
- 月度复盘及行动状态。

建议把文件保存到手机文件夹、电脑或可信的个人云盘。软件会在长期未备份、增加较多记录或配置发生变化时提醒备份。

### 导入备份

选择 JSON 文件后，软件会先展示文件格式、记录数量、日期范围、支出合计、历史收入和可识别的规划数据。

默认使用“完整恢复账目与设置”：

- 恢复文件中的账目、设置和规划；
- 旧备份缺少的设置或规划会保留当前数据；
- 执行前会自动下载当前完整备份。

“高级导入方式”还提供：

- “合并账目／保留当前数据”：按记录 ID 去重后加入当前账本，不修改设置和规划；
- “覆盖账目／设置保持不变”：用文件中的账目替换当前账目，不修改设置和规划。

导入完成后，应检查总记录数、最近几笔账目、预算、专项和目标是否符合预期。

### 数据异常救援

如果本地数据无法完整解析，软件会暂停新增、编辑和删除，避免覆盖原始内容。此时按照“数据救援”提示：

1. 先下载未经处理的原始数据；
2. 确认文件已经保存；
3. 再选择保留当前能够识别的有效记录。

## 推荐使用节奏

- 每天：随手记录支出。
- 每周：检查明细，补充漏记，观察预算消耗。
- 发生旅行、装修等事件前：建立正式专项并设置独立预算。
- 每月结束后：完成月结，只保留一个结论和一个下月行动。
- 每次重要调整后：导出完整备份。

## 演示数据

仓库中的 `test-data/account-book-demo-v2.json` 是用于界面测试的演示账本。导入前先备份自己的真实数据，并在导入预览中确认操作方式。

## 当前边界

- 没有登录、服务端同步或多人实时协作。
- 没有银行账户连接、自动抓取账单或资产负债管理。
- 没有自动家庭汇总；每位成员仍使用各自的本地账本。
- 支出分类目前是系统内置分类，暂不支持用户自行增删。

项目进度和后续计划见 `ROADMAP.md`。功能、导航、字段或操作流程发生变化时，本说明必须同步更新。
