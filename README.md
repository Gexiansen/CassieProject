# 我的小账本

一个通过 GitHub Pages 使用的手机端本地支出记账工具。它通过“支出—预算—目标—月结”帮助个人或家庭成员在各自设备上做财务决策。

主账本入口是 `docs/index.html`。`allen_account.html` 是独立的儿童玩具钱包，不与主账本共享数据，本说明不涵盖该页面。

## 使用前必须知道

- 软件没有账号、服务器和网络同步，账目、设置、预算、专项、目标及月结都保存在当前浏览器中。
- 同一部手机上的不同浏览器、普通模式与无痕模式拥有不同的数据空间。
- 清除网站数据、卸载浏览器或更换设备，可能导致本地数据丢失。
- 请定期通过右上角“数据管理”导出完整 JSON 备份，重要操作前也建议先备份。
- 后续即使家庭成员多人使用，也优先由每个人在自己的设备上独立记账，再通过备份文件迁移；当前没有自动合并家庭账本的功能。
- 当前正式数据格式是 schema v4。页面不会读取、覆盖或删除旧版本地数据；检测到旧数据时，必须导入转换后的 v4 完整备份，或明确选择从空白 v4 账本开始。

## 首次使用

1. 用手机浏览器打开部署在 GitHub Pages 上的页面。
2. 如果页面提示检测到旧版账本，在“数据管理”中导入转换后的 v4 完整备份，或确认开始空白 v4 账本。
3. 在首页点击今天的日期，输入金额后点击分类，记录第一笔支出；也可以使用右下角“＋ 记一笔”。
4. 进入“计划—预算”，设置本月可支配金额和日常预算。
5. 使用一段时间后，在“计划—月结”查看本月结果并写下一个结论和下个月的一件事。
6. 点击右上角“数据”，导出第一份完整备份。

首次成功打开后，页面支持离线使用。为了更接近原生应用体验，可以在手机浏览器菜单中选择“添加到主屏幕”或“安装应用”。

## 页面结构

软件只有三个一级入口：

- “首页”：通过默认显示一周的日历快速记账，并查看本月支出、可支配金额、预算进度、当前专项、消费观察、支出趋势和排行。
- “明细”：用列表查看账目，并进行搜索、筛选、编辑、复制和删除。
- “计划”：管理预算、正式专项、财务目标和月结。

首页日历的左右箭头在周视图中切换上一周／下一周，展开整月后切换上个月／下个月。明细页仍可切换月／年统计周期；当前月份和年份只与上月同期或上年同期比较，避免拿不完整周期与完整周期直接比较。

## 记录一笔支出

点击首页没有支出记录的日期，或点击任意页面右下角“＋ 记一笔”：

1. 输入金额。
2. 备注与金额在同一行，可按需填写。
3. 顶部显示当前启用的家庭获益方；存在适用于当前日期的当前专项时，专项会排在成员之后并默认选中。
4. 点击“最近一笔”、当前成员的本地高频分类或“全部分类”中的任意细分类，支出会立即保存；全部分类使用纯文字，按大类标题纵向分组，不需要先选择大类。
5. “最近一笔”只从当前成员或专项的记录中实时计算，并会在备注栏为空时沿用该笔备注；手动输入的备注优先。本地高频分类只复用分类，不自动带入历史备注。
6. 如需修改日期或手动关联其他专项，展开“日期与专项”。

新增支出的主路径是“输入金额—点击分类即保存”，不再需要额外点击保存按钮。编辑已有支出时仍需点击“保存修改”。软件不再逐笔记录收入，金额在内部按整数分保存，界面统一按元显示。

保存后会显示 12 秒“撤销”入口。点击已有记录的日期，会先显示当日支出总额和明细；点击“＋ 再记一笔”可以继续补记该日支出。

### 当前专项

存在“当前专项”且记账日期位于专项开始和结束日期之间时，专项会排在快速记账顶部最后并默认选中。点击任一家庭成员，可把本次支出改回日常支出；专项日期之外不会自动关联，但仍可在“日期与专项”中手动选择。

### 家庭成员管理

点击右上角“数据—家庭成员”，可以新增、改名、排序、停用和恢复成员，并设置本机默认获益方。“共同”是保留项，不能停用；最多同时启用 8 个获益方。1～4 个获益方显示一行，5～8 个分两行显示，不使用横向滑动。成员顺序和默认项只保存在本机，并随完整备份迁移。

### 分类管理

点击右上角“数据—分类管理”，可以在本机调整支出分类：

- 新增、改名和调整大类或细分类顺序；
- 指定每个细分类显示在哪些家庭成员页；
- 停用暂时不用的分类，之后仍可恢复。

分类使用全局唯一且稳定的细分类 ID 保存。改名、排序或停用不会删除已有账目，也不会改变历史记录的金额；分类设置只保存在当前浏览器，并随完整备份迁移。

## 查看和管理明细

进入“明细”后，可以按年份和月份查看账目列表。

### 列表

- 账目默认按年份和月份折叠汇总。
- 点击年份或月份可以展开和收起。
- 每笔支出右侧的“⋯”提供复制、编辑和删除操作。
- 删除后有 12 秒撤销时间。

### 搜索和筛选

搜索框可以查找备注、专项名称、获益方或分类。展开“更多筛选条件”后，还可以组合筛选：

- 全部、本月、上月、今年或自定义日期。

筛选结果会显示记录数和支出汇总。

### 首页日历

- 首页默认只显示包含当前日期的一周，点击“••• 展开整月”可以查看整月，再点击“收起到本周”恢复周视图。
- 紫色表示当天已有支出记录，并显示当日支出合计；浅黄色表示过去日期没有记录；绿色表示已经主动确认当日无支出；未来日期使用灰色。
- “无记录”只表示软件中没有该日支出，不能直接判断用户是否漏记。
- 点击没有记录的今天或过去日期，会直接打开该日期的快速记账；也可以点击“确认当日无支出”。
- 点击已有记录的日期，会先显示当天总额和每笔支出，再继续记账或管理已有记录。
- 新增支出后，对应日期会自动变为已记录；如果此前确认无支出，该确认会自动取消。

## 设置日常预算

进入“计划—预算”：

1. 可选填写“本月可支配金额”，表示本月可以安排给支出和目标的钱。
2. 设置“本月日常预算”。
3. 点击“保存月度计划”。

如果上个月已经设置月度计划，可以复制上月的可支配金额和日常预算作为起点。

正式专项支出不会占用日常总预算。例如旅行期间的住宿、交通和餐饮仍保留原分类，但由旅行专项单独归集。

预算页会用本月可支配金额减去日常预算、本月开始的专项预算和进行中目标的建议投入，给出一个“计划后结余”或缺口。没有设置日常预算时，才会使用最近三个完整月份的历史平均支出作为参考；缺少可支配金额时不会给出结余判断。

## 管理正式专项

正式专项用于记录一次完整的生活事件，例如旅行、装修、节日、医疗、教育或搬家。

进入“计划—专项”，点击“＋ 新建专项”，填写：

- 专项名称和类型；
- 专项预算，选填；
- 开始和结束日期；
- 旅行参与人数，旅行专项必填，仅旅行专项显示。

新建专项时，如果已有同类型且包含支出记录的已完成专项，表单会展示最近一次的实际支出、当时预算、主要支出分类和周期。旅行还会显示参与人数、人均及人均每天费用。历史数据只作参考，不会自动填写或修改本次预算。

首个进行中的专项会自动成为当前专项。也可以在专项卡片中手动“设为当前”或“取消当前”。完成专项后，它不会再自动关联新支出。

专项页面会展示实际支出和记录数；设置预算后还会展示预算执行和剩余金额。旅行专项会计算人均和人均每天费用，便于以后规划类似旅行。存在其他同类型历史专项时，可以展开“对比最近 N 次”查看最近五次已完成且包含支出的专项，比较实际支出、预算执行、主要分类、周期，以及旅行的人均费用。支出明细可以直接点击管理，已完成专项默认折叠显示。

## 管理财务目标

进入“计划—目标”，可以建立应急储备、旅行、教育、偿债或自定义目标。

每个目标包含目标金额和目标日期，系统会根据剩余金额与剩余月份计算建议月投入。

目标只表示为未来留出资金；旅行、装修等实际花费应使用专项。当你实际留出一笔钱后，点击目标中的“记录投入”进行确认。目标投入：

- 不会自动生成支出记录；
- 可以查看、编辑和撤销每笔投入；
- 会同步影响目标进度和对应月份的月结汇总。

目标支持暂停、恢复、完成和重新启用。编辑、状态切换和投入记录纠正集中在“管理”中，已暂停和已完成目标默认折叠显示。

## 完成月结

进入“计划—月结”，可以查看：

- 本月可支配金额、支出和剩余金额；
- 日常预算执行情况；
- 目标投入汇总；
- 基于当前本地数据生成的本月观察。

月结只要求填写两项：

1. 本月最值得记住的一个发现；
2. 下个月只做的一件事。

上个月保存的行动会在下个月出现，并可以标记完成。未来月份不能提前填写月结。

## 备份和恢复数据

点击页面右上角“数据”进入“数据管理”。

### 导出完整备份

“导出完整备份”会生成 JSON 文件，包含：

- 全部支出记录及获益方；
- 家庭成员名称、顺序、启用状态和本机默认获益方；
- 分类名称、顺序、启用状态和成员显示范围；
- 月度预算；
- 正式专项及当前专项；
- 财务目标和投入记录；
- 月度复盘及行动状态；
- 已确认无支出的日期。

建议把文件保存到手机文件夹、电脑或可信的个人云盘。软件会在长期未备份、增加较多记录或配置发生变化时提醒备份。

### 导入备份

正式页面只接受完整备份 v4。选择文件后，软件会整体校验账目、家庭成员、分类、预算、专项、目标、月结和引用关系；任何无效项都会拒绝恢复，不会静默丢弃记录。

预览会显示文件名、导出时间、日期范围、记录数、支出合计、家庭成员、分类、专项和目标数量。确认后会完整替换当前 v4 账目、家庭设置和规划；当前已有 v4 数据时，执行前会自动下载完整备份。v4 不提供部分覆盖或自动合并，避免分类、成员和规划引用被拆散。

导入完成后，应检查总记录数、最近几笔账目、预算、专项和目标是否符合预期。

### 从 v3 或更早版本升级

检测到旧版本地数据时，页面会暂停写入并给出两个选择：

1. 先导出旧备份，在电脑中运行一次性转换工具，再导入生成的 v4 完整备份；
2. 明确确认保留旧数据并开始空白 v4 账本。

转换工具不会被网页加载，也不会读取浏览器数据。命令示例：

```bash
node tools/convert-v3-to-v4.mjs old-backup.json new-v4-backup.json --unassigned family
```

`--unassigned` 用于明确指定旧记录中缺少获益方的数据，可选值是 `family`、`wife`、`husband` 或 `son`。工具会报告原始记录数、支出数、排除的收入数、支出总额、日期范围、旧专项标签及所有未映射项；只有引用全部有效且转换前后支出总额一致时才生成文件，已有输出文件不会被覆盖。

### 数据异常救援

如果本地数据无法完整解析，软件会暂停新增、编辑和删除，避免覆盖原始内容。此时按照“数据救援”提示：

1. 先下载未经处理的原始数据；
2. 确认文件已经保存；
3. 再选择保留当前能够识别的有效记录。

## 推荐使用节奏

- 每天：从首页日历记录支出；确实没有支出时，可以确认“当日无支出”。
- 每周：查看周历中的无记录日期，补充漏记，再观察预算消耗。
- 发生旅行、装修等事件前：建立正式专项并设置独立预算。
- 每月结束后：完成月结，只保留一个结论和一个下月行动。
- 每次重要调整后：导出完整备份。

## 演示数据

仓库中的 `test-data/account-book-demo-v4.json` 是可直接导入当前软件的 v4 演示账本，包含 34 笔支出。`test-data/account-book-demo-v2.json` 只作为旧格式转换测试输入，不能直接导入正式页面。

## 当前边界

- 没有登录、服务端同步或多人实时协作。
- 没有银行账户连接、自动抓取账单或资产负债管理。
- 没有自动家庭汇总；每位成员仍使用各自的本地账本。
- 家庭成员和分类设置只在本机生效；其他设备需要通过完整备份迁移。
- v4 不自动解析或合并旧备份，旧格式必须先通过一次性工具转换。

项目进度和后续计划见 `ROADMAP.md`。功能、导航、字段或操作流程发生变化时，本说明必须同步更新。
