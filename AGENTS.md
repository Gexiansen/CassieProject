# CassieProject 项目规范

## 项目定位

- `docs/index.html` 是通过 GitHub Pages 使用的手机端日常支出账本和静态站点入口。
- `allen_account.html` 是独立的儿童玩具钱包，不与主账本共享数据模型。
- 主账本坚持本地优先：数据保存在各自浏览器中，通过 JSON 文件备份和迁移，不规划登录、服务端同步和家庭权限。
- 后续即使多人使用，也优先保持每位成员各自在本地使用；只有明确需要汇总时再设计手动文件合并。

## 技术约定

- 当前保持原生 HTML、CSS、JavaScript，不引入前端框架或第三方运行时依赖。
- `docs/` 是 GitHub Pages 发布目录，必须保持相对路径、离线可用且不依赖第三方网络资源。
- `tools/` 只放不被正式页面加载的一次性离线工具；`tests/` 使用 Node.js 标准库验证数据模型和转换流程；`test-data/` 只放演示或测试数据，禁止加入真实用户备份。
- 金额在存储和计算中统一使用整数分，界面层负责元与分的转换。
- 业务记录使用 UUID，不使用时间戳作为唯一标识。
- 所有持久化数据必须带结构版本，并在读取、导入时完成校验和迁移。
- 用户或导入数据不得直接拼入内联事件处理器；交互统一通过事件监听和 `data-*` 属性绑定。
- 修改数据模型时必须同时考虑旧数据迁移和导入导出兼容性。

## 修改原则

- 只修改当前任务涉及的文件和逻辑，不顺手改动 `allen_account.html`。
- 业务计算尽量保持为可独立验证的纯函数。
- 数据保存失败必须向用户显示，不得静默忽略。
- 删除账目默认可撤销；共享阶段再演进为软删除和审计。
- 完成开发后必须运行 JavaScript 语法检查，并验证新增、编辑、删除撤销、导入和旧数据迁移流程。

## 文档维护

- `README.md` 用于项目介绍和使用方法。
- 功能、导航、表单字段、数据管理方式或用户操作流程发生变化时，必须在同一次修改中同步更新 `README.md`。
- `README.md` 只描述已经实现并验证过的功能，界面名称和操作路径必须与 `docs/index.html` 保持一致；纯内部重构且不影响使用方式时无需更新。
- `ROADMAP.md` 是进度事实源；已实现且验证通过后才能标记完成。
- 纯讨论和只读分析不更新路线图；代码或项目状态发生变化后必须同步更新。
